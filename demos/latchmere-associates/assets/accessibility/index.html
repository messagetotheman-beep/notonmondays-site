Phase 1 — homepage wireframe

Item Status Notes
Skip link to #mainPassPresent and functional. Becomes visible on focus.
Single h1 per pagePassh1 in hero only. h2 for sections, h3 for sub-sections.
Heading hierarchyPassNo skipped levels. h1 → h2 → h3 throughout.
Landmark regionsPass<header>, <main>, <footer>, <nav> all present.
aria-labelledby on sectionsPassAll major sections labelled by their heading id.
Nav aria-labelPassMain nav and footer nav both carry aria-label.
Decorative emoji aria-hiddenPassAll card icons and decorative elements marked aria-hidden="true".
Focus-visible stylesPass3px #ffbf47 outline on all interactive elements. Meets WCAG 2.4.11.
lang attribute on htmlPasslang="en" set.
Semantic list for service cardsPassrole="list" and role="listitem" used. Cards are links.
Steps as ordered listPassHow we help and divorce process both use <ol>.
Testimonials use blockquote/citePassSemantic quote markup used throughout.
Colour contrast — body textPass#1a1a1a on white. Ratio ~18:1. Exceeds AA.
Colour contrast — muted textPass#5a5a5a on white. Ratio ~7:1. Meets AA.
Colour contrast — white on navyPassRatio ~15:1. Exceeds AA.
Colour contrast — white on tealPass#ffffff on #1a7a6e. Ratio ~5.2:1. Meets AA for normal text.
Annotation elements in DOM flowNoteAnnotation spans are read by screen readers between sections. Acceptable for a prototype; should be removed or marked role="note" before production.
Mobile tap target sizePassAll buttons and card links exceed 44×44px on mobile. Meets WCAG 2.5.5.
prefers-reduced-motionNoteNo animations in Phase 1. Will need a reduced-motion rule if transitions are added in later phases.
Form elementsNoteNo forms in Phase 1. Contact and enquiry forms will need full label, error, and live region testing in Phase 2.

Open items for Phase 2

  • Remove or properly mark prototype annotations before production
  • Test with a screen reader (VoiceOver / NVDA) once forms are added
  • Audit all images and media added in later phases (alt text, captions)
  • Add prefers-reduced-motion rule if any CSS transitions are introduced
  • Test keyboard navigation end-to-end on all page templates
